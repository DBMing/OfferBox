# OfferBox · 投递旅程 🎯

> 一个专注于招聘投递流程管理的极简网页应用，帮助你优雅地管理求职全流程。

从投递简历到收获 Offer，OfferBox 让你随时掌握每个关键节点。它提供双核心模块：**投递记录**追踪你的求职进度，**招聘信息库**收集心仪职位。所有数据持久化保存在本地 JSON 文件中，完全离线可用，数据隐私完全掌控。

## ✨ 核心特性

### 🎨 视觉设计
- **苹果风格的轻盈美学**：淡蓝与淡紫的渐变背景、玻璃拟态卡片与细腻动效
- **智能状态视觉反馈**：
  - 🎉 成功 Offer：绿金渐变 + 庆祝动画
  - 📝 进行中：优雅的蓝紫色系
  - 😢 已结束：低调的灰色调传达失败情绪

### 📊 双核心模块
- **投递记录**：
  - 自定义流程阶段（投递 → 初筛 → 面试 → Offer）
  - 可交互流程箭头，点击更新当前阶段
  - 四维统计面板：正在跟进/成功 Offer/已结束/最新更新
  - 结束投递功能，标记失败的投递
  - 网申进度按钮，快速访问投递链接

- **招聘信息库**：
  - 收集心仪职位信息
  - 优先级标记（高/中/低）
  - 兴趣度评分（五星评级）
  - 匹配度评估
  - 一键转为投递记录

### 💡 交互功能
- **列表 + 聚焦卡片双视图**：列表浏览关键信息，点击弹出带背景虚化的详情卡片
- **Markdown 备注系统**：详情卡片提供可滚动的 Markdown 区域，记录面试纪要与准备要点
- **实时数量统计**：左侧导航显示各模块记录数量徽章
- **持久化存储**：数据保存在本地 JSON 文件，支持完整 CRUD 操作

## 🚀 快速开始

### 环境要求
- Node.js ≥ 16
- 无需安装任何第三方 npm 包（项目使用原生 Node.js HTTP 模块）

### 安装运行

```bash
# 1. 克隆项目
git clone https://github.com/yourusername/OfferBox.git
cd OfferBox

# 2. （可选）安装依赖，会生成 package-lock.json
npm install

# 3. 启动本地服务
npm start
```

服务默认运行在 [http://localhost:3000](http://localhost:3000)。

首次打开数据为空，你可以直接在界面内进行操作，所有改动都会实时保存到本地 JSON 文件。

**数据隐私保护：**
- 你的投递记录和招聘信息保存在 `server/data/` 目录下的 JSON 文件中
- 项目初次克隆时包含空的数据文件，之后你的数据修改**不会被 git 跟踪**
- 你的个人数据完全保存在本地，隐私安全有保障

### 停止服务

**方法一：终端中停止**
- 在运行 `npm start` 的终端窗口中按 `Ctrl + C`（Windows/Linux）或 `Cmd + C`（macOS）

**方法二：查找并关闭进程**

```bash
# 查找运行中的进程
lsof -i :3000

# 或使用
ps aux | grep "node.*server/index.js"

# 关闭进程（将 PID 替换为实际进程 ID）
kill <PID>

# 或强制关闭
kill -9 <PID>
```

**方法三：一键关闭（macOS/Linux）**

```bash
# 关闭所有运行在 3000 端口的进程
lsof -ti :3000 | xargs kill

# 或关闭所有 OfferBox 服务进程
pkill -f "node.*server/index.js"
```

## 📖 使用指南

### 投递记录管理

#### 1️⃣ 新增投递记录
1. 点击右上角 **"+ 新增投递"** 按钮
2. 填写必填信息：
   - **公司名称**、**职位名称**（必填）
   - 工作地点、工作性质、投递日期
   - 薪资预期、网申链接
3. **流程阶段设置**：
   - 默认流程：投递 → 初筛 → 一面 → 二面 → 三面 → HR面 → Offer
   - 可自由添加或删除阶段，输入新阶段名称后点击"+ 添加阶段"
   - 点击阶段标签上的 × 可删除该阶段
4. 选择当前所处阶段
5. 添加备注（支持 Markdown 格式）
6. 点击 **"保存"** 完成创建

#### 2️⃣ 查看投递详情
- 点击列表中的任意投递记录卡片
- 详情页显示：
  - 流程进度条（点击可快速更新阶段）
  - 完整投递信息
  - Markdown 格式的备注内容
  - 操作按钮：编辑、结束投递、删除

#### 3️⃣ 更新投递进度
**方式一：详情页流程条**
- 打开投递详情
- 点击流程进度条上的任意阶段节点
- 系统自动保存进度

**方式二：编辑表单**
- 点击详情页右上角 **"编辑"** 按钮
- 在"当前流程阶段"下拉框中选择新阶段
- 点击 **"保存"** 更新

#### 4️⃣ 结束失败的投递 😢
- 打开投递详情
- 点击右上角 **"结束投递"** 按钮
- 确认后，该记录会：
  - 在列表中显示为灰色调
  - 从"正在跟进"移入"已结束"统计
  - 无法再次标记为结束

#### 5️⃣ 庆祝成功 Offer 🎉
- 将投递记录的流程阶段更新到最后一个（默认为 "Offer"）
- 卡片自动变为绿金渐变 + 庆祝动画
- 统计面板"成功 Offer"数量 +1

#### 6️⃣ 访问网申链接
- 投递记录卡片上的 **"网申进度"** 按钮可直接跳转到网申链接
- 如果未填写网申链接，按钮会显示为禁用状态

### 招聘信息库管理

#### 1️⃣ 添加招聘信息
1. 切换到左侧导航栏的 **"招聘信息库"**
2. 点击右上角 **"+ 新增职位"** 按钮
3. 填写职位信息：
   - 基本信息：公司名称、职位名称、工作地点等
   - 信息来源（如：拉勾网、BOSS直聘等）
   - 网申链接、截止日期
4. 设置评估指标：
   - **优先级**：高/中/低（用于排序）
   - **兴趣度**：1-5 星评分
   - **匹配度**：1-5 星评分
5. 填写职位描述和要求（支持 Markdown）
6. 点击 **"保存"**

#### 2️⃣ 一键网申
- 在招聘信息卡片上点击 **"一键网申"** 按钮
- 直接在新标签页打开该职位的网申链接

#### 3️⃣ 一键转为投递记录
- 点击招聘信息卡片上的 **"投递"** 按钮
- 确认后自动：
  - 创建对应的投递记录
  - 同步所有基本信息（公司、职位、网申链接等）
  - 卡片按钮变为 **"已投递"**（禁用状态）
  - 自动跳转到投递记录详情页

### 统计面板说明

**投递记录统计**（顶部四格面板）：
- **正在跟进**：未结束且未到达最后阶段的投递数量
- **成功 Offer**：到达最后阶段（Offer）的投递数量
- **已结束**：手动标记为失败的投递数量
- **最新更新**：最近一次更新记录的时间

**招聘信息统计**（顶部两格面板）：
- **收藏职位**：招聘信息库中的总职位数
- **最新添加**：最近一次添加职位的时间

## 🔧 高级技巧

### 自定义流程阶段
- 在新增/编辑投递记录时，可以自由修改流程阶段
- 支持添加任意数量的自定义阶段（如：OA测试、技术面、产品面等）
- 流程进度条会自动适配显示所有阶段

### Markdown 备注技巧
备注框完全支持 Markdown 语法，可以这样使用：

```markdown
## 面试准备
- [x] 复习项目亮点
- [ ] 准备自我介绍
- [ ] 研究公司业务

## 面试记录
**一面（技术面）- 2024-03-10**
- 问题1：项目中的难点？
- 问题2：如何优化性能？

**笔记**
> 面试官很看重实际项目经验
```

### 数据备份与迁移
- **备份**：复制 `server/data/` 目录下的 JSON 文件
- **迁移**：将备份的 JSON 文件粘贴到新环境的 `server/data/` 目录
- **导出**：JSON 文件可直接用文本编辑器打开，方便做批量处理

### 修改服务器端口
编辑 `server/index.js` 第一行：

```javascript
const PORT = process.env.PORT || 3000;  // 修改为你想要的端口号
```

## 📝 API 文档

### 投递记录 API

**获取所有投递记录**
```http
GET /api/applications
```

**创建投递记录**
```http
POST /api/applications
Content-Type: application/json

{
  "company": "示例公司",
  "position": "前端工程师",
  "location": "北京",
  "jobType": "全职",
  "appliedDate": "2024-03-15",
  "salaryRange": "20-30K * 14",
  "sourceUrl": "https://...",
  "stages": ["投递", "初筛", "一面", "二面", "Offer"],
  "currentStage": 0,
  "notes": "备注内容"
}
```

**更新投递记录**
```http
PATCH /api/applications/:id
Content-Type: application/json

{
  "currentStage": 2,
  "notes": "更新的备注"
}
```

**删除投递记录**
```http
DELETE /api/applications/:id
```

**结束投递**
```http
PATCH /api/applications/:id
Content-Type: application/json

{
  "isTerminated": true,
  "terminatedAt": "2024-03-20T10:00:00.000Z"
}
```

### 招聘信息 API

**获取所有招聘信息**
```http
GET /api/jobs
```

**创建招聘信息**
```http
POST /api/jobs
Content-Type: application/json

{
  "company": "示例公司",
  "position": "产品经理",
  "location": "上海",
  "source": "拉勾网",
  "sourceUrl": "https://...",
  "priority": "high",
  "interestLevel": 4,
  "matchScore": 5,
  "description": "职位描述",
  "notes": "备注"
}
```

**一键转为投递记录**
```http
POST /api/jobs/:id/apply
```
